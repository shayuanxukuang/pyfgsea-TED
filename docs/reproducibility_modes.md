# Reproducibility Modes

TED should be distributed with three reproduction modes so reviewers do not need to rerun every large public dataset.

## Mode 1: minimal demo

- Uses small synthetic or toy data.
- Target runtime: less than 30 minutes on a standard workstation.
- Purpose: verify installation, event object construction, claim ceiling logic and output schema.
- Suggested command: `python reproducibility/run_minimal_demo.py`

## Mode 2: main figure reproduction

- Uses processed TSV/H5AD/matrix outputs generated by the analysis pipeline.
- Purpose: reproduce main figures, final claim matrix and manuscript tables.
- Suggested commands:
  - `python reproduce_all_main_tables.py`
  - `python reproduce_all_main_figures.py`

## Mode 3: full analysis

- Starts from downloaded public data and regenerates major TED outputs.
- Purpose: complete computational audit.
- Expected to require substantially more time and storage.
- Suggested command family: dataset-specific scripts under `scripts/`.

## Required smoke checks before submission

```bash
python -m py_compile scripts/*.py
pytest -q
ruff check
python reproduce_all_main_tables.py
python reproduce_all_main_figures.py
```

If a check is not run before submission, record it explicitly in the reproducibility checklist rather than leaving the status implicit.
